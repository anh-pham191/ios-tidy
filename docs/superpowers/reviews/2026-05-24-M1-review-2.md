## New findings (cycle 2)

### High

- **[H/new — N1] Plan drifts from the SHARED_CONTEXT.md §11 camelCase JSON lock-in.** The cycle-1 reviewer raised JSON-key convention as Medium (M4) because no convention was pinned. Between cycles, SHARED_CONTEXT.md §11 was added (dated 2026-05-24) locking JSON keys to camelCase: `totalBytes`, `freeBytes`, `bundleID`, `dynamicBytes`, `staticBytes`. The revised plan does not honour this:
  - `cmd/ios-tidy/devices.go` (Task 7a Step 3, lines 1297–1303 of the plan):
    ```go
    UDID       string `json:"udid"`
    Name       string `json:"name"`
    Model      string `json:"model"`
    IOSVersion string `json:"ios_version"`
    Trusted    bool   `json:"trusted"`
    ```
    `"ios_version"` is snake_case. The §11 convention requires `"iosVersion"`. By the §11 examples (`bundleID`) the UDID tag should arguably be `"udid"` (already lowercase, no word break — fine) but `IOSVersion` clearly maps to `"iosVersion"`.
  - `cmd/ios-tidy/devices_test.go` (Task 7b Step 5): asserts `` `"ios_version": "18.4"` `` — same drift.
  - Open question #6 documents and *defends* the snake_case choice ("dominant Unix-CLI JSON convention … grep-friendly … M2/M5 inherit"). With §11 in place this OQ should be marked **resolved to camelCase** with the rationale linked back to §11, not left as an unresolved snake_case proposal.

  Severity: **High** — drift from a load-bearing decision that M2 (`storage --json`) and M5 (`apps probe --json`) inherit. Catching it here is exactly the kind of public-surface fix that is cheap pre-execution and expensive post-shipping.

  Required edit (minimal, three locations):
  1. `cmd/ios-tidy/devices.go` `deviceRow` JSON tags: `"udid"`, `"name"`, `"model"`, `"iosVersion"`, `"trusted"`. Update the doc-comment above `deviceRow` to point at SHARED_CONTEXT.md §11.
  2. `cmd/ios-tidy/devices_test.go` `TestDevicesCmd_jsonOutputUsesTrustedBool`: change `` `"ios_version": "18.4"` `` to `` `"iosVersion": "18.4"` ``.
  3. Open question #6: rewrite to reflect the §11 lock-in (or remove and replace with a one-line "JSON keys are camelCase per SHARED_CONTEXT.md §11").
