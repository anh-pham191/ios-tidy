## Conclusion

**Sign-off granted.** Execute the plan task-by-task.

The single most important issue to keep in mind during execution (user-data safety): **the placement of the `if *dryRun { return 0 }` block must stay between `ui.RenderCleanPlan(...)` and the `if *includeDocs { ... } else { ... }` prompt branch.** Task 11 Step 3 establishes this placement, Step 9 re-asserts it, and `TestAppsClean_dryRunWithDocumentsNeverCallsRemoveOrPrompts` (Step 7) is the regression net. If a future refactor relocates the short-circuit below the Documents branch, the typed-bundle-ID prompt would fire under `--dry-run` and the user could be coerced into typing a confirmation that does nothing — eroding the dry-run trust contract. The test catches it; do not modify or move the test.
